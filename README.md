# Single-Agent Tool-Based ML Demo

This demo uses a clearer structure: the LLM helps with planning, target/task inference, tuning guidance, and business reporting, while deterministic Python tools execute data intake, cleaning, EDA, feature engineering, model training, and validation.

The goal is to make the workflow more stable, auditable, and easier to debug.

## Demo Structure

```text
single_agent/
  main.py                     # Sequential workflow runner and CLI entry point

  agents/
    manager.py                # Dataset intake, target/task inference, plan creation, replanning
    data_cleaning.py          # Shared cleaning and train/test preparation
    eda_analysis.py           # Leakage-aware EDA
    feature_decision.py       # Feature engineering policy decision
    feature_engineering.py    # Model-specific feature pipelines
    model_selection.py        # Candidate model tuning, evaluation, and selection
    business_translator.py    # Business-facing markdown report
    supervisor.py             # Deterministic artifact validation

  tools/
    data_intake_tools.py
    data_cleaning_tools.py
    eda_tools.py
    feature_engineering_tools.py
    model_policy_tools.py
    model_selection_tools.py
    model_tuning_tools.py
    target_inference_tools.py
    validation_tools.py
    registry.py

  core/
    state_schema.py           # Shared state and artifact paths
    llm.py                    # OpenAI-compatible LLM client
    progress.py               # Runtime progress logging

  data/                       # Example datasets
  docs/                       # Architecture notes and workflow graph
  workspace/                  # Runtime outputs generated by the workflow
```

## Workflow

```text
Manager
-> Data Cleaning
-> Supervisor
-> Manager Replanning Checkpoint
-> EDA Analysis
-> Supervisor
-> Feature Decision
-> Feature Engineering
-> Supervisor
-> Manager Replanning Checkpoint
-> Model Selection
-> Supervisor
-> Business Translator
-> Supervisor
```

Key artifacts are written to `workspace/`, including:

- `manager_plan.json`
- `plan_patches.json`
- `cleaning_report.json`
- `eda_report.json`
- `feature_policy_report.json`
- `feature_report.json`
- `model_report.json`
- `feature_importance.json`
- `business_report.md`

## Quickstart

### 1. Create and activate an environment

```bash
python -m venv .venv
source .venv/bin/activate
```

### 2. Install dependencies

```bash
pip install pandas numpy scikit-learn
```

Optional: configure an OpenAI-compatible LLM for better target/task inference, tuning explanations, and business report generation.

```bash
cp .env.example .env
```

Then edit `.env`:

```text
OPENAI_API_KEY=your_api_key_here
OPENAI_MODEL=gpt-4o-mini
LLM_TEMPERATURE=0.2
```

If no LLM is configured, the workflow still uses deterministic fallbacks where possible.

### 3. Run a single-file dataset

The workflow will read one CSV and create its own train/test split.

```bash
python main.py \
  --data data/bank-full.csv \
  --goal "Predict whether the client will subscribe a term deposit" \
  --description-file data/bank-names.txt
```

You can also provide a short description directly:

```bash
python main.py \
  --data data/Financials.csv \
  --goal "Predict business-relevant financial outcomes and generate actionable recommendations" \
  --description-text "Financial sales dataset. Profit is the supervised regression target."
```

### 4. Run provided train/test files

Use this mode when the dataset already has separate train and test files.

```bash
python main.py \
  --train path/to/train.csv \
  --test path/to/test.csv \
  --goal "Predict customer churn" \
  --description-file path/to/data_description.txt
```

Current limitation: the provided test file must include the target column, because the workflow evaluates final test metrics. Prediction-only test files are not supported yet.

### 5. Review outputs

After a successful run:

```text
Workflow completed.
Plan version: <number>
Business report: workspace/business_report.md
Artifacts written to ./workspace
```

Open the final report:

```bash
less workspace/business_report.md
```

Inspect structured outputs:

```bash
cat workspace/manager_plan.json
cat workspace/model_report.json
cat workspace/eda_report.json
```

## Notes

- EDA avoids data leakage by using `train_data.csv` only for target and feature-target relationship analysis.
- Feature engineering fits transformations on train data and applies compatible transforms to test data.
- Manager replanning is controlled and checkpoint-based. It writes versioned patches to `workspace/plan_patches.json`.
- Supervisor validation is deterministic and checks required artifacts before the workflow continues.
